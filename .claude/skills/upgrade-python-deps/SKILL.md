## Your task

Upgrade all Python dependencies and verify nothing is broken.

### Step 1: Upgrade the lock file

Run `uv lock --upgrade` to upgrade all dependencies to their latest compatible versions
allowed by `pyproject.toml`.

Review the output for any resolution errors. If there are conflicts, report them to the
user and ask how to proceed before continuing.

Call out **major version bumps** explicitly — Django, Celery, allauth, Ruff, and pytest
all change behaviour across major versions and may require code or settings updates.

### Step 2: Sync the environment

Run `uv sync` to install the upgraded dependencies into the virtual environment.

### Step 3: Run post-upgrade checks

Run these checks sequentially, stopping if any step fails:

1. **Django system check**: Run `uv run manage.py check` to surface any deprecation
   warnings or app-loading issues introduced by the upgraded packages.
2. **Migration drift**: Run `uv run manage.py makemigrations --check --dry-run`. Some
   package upgrades (notably `django`, `django-allauth`, `django-celery-beat`) ship new
   model fields. If the command reports pending migrations, stop and report — the user
   will need to decide whether to commit them as part of this upgrade or separately.
3. **Lint**: Run `make lint`. Ruff bumps frequently introduce new rules; report any new
   findings rather than auto-fixing them blindly. djlint may also report new template
   issues.
4. **Tests**: Run `make test` to verify the test suite still passes. Watch for new
   `DeprecationWarning`s in the output — they often signal upcoming breakage in the
   next major version.

### Step 4: Summarize

Summarize what was done:

- Which packages were upgraded (group by major / minor / patch).
- Whether the Django check passed and any deprecation warnings raised.
- Whether any new migrations were generated by the upgrade.
- Whether all tests passed and any new deprecation warnings observed.
- Any new lint findings (and whether they were auto-fixed or need manual attention).
- Any issues that need manual follow-up.

If there were failures, present the issues and ask how the user wants to proceed.

If everything passed, ask the user if they'd like to commit the changes. If yes, hand off
to the `commit` skill — suggest a subject line like `chore(deps): upgrade Python dependencies`.
